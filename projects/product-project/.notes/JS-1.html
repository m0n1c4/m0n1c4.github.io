JavaScript Variables

In JavaScript you need to precede a new variable with var:

1
2
 
// JavaScript
var name = 'Bob';

Notice how you need a semicolon at the end. This means "end of statement."

The var keyword is used for declaring a new variable. If you want to update an existing variable, leave it out:

1
2
3
4
 
// JavaScript
var score = 11;
score = 5;
score = score + 1;

Variables, Strings, Data, and Code

What's the difference between a variable and a string? Well, as you know, aesthetically strings have quotes while variables do not:

1
2
 
'person' // STRING
person // VARIABLE

Semantically, you could say a string is data, while a variable is code.

What does that mean? Let's take it by example. Consider the following real-world things:

  • A single playing card
  • The current price on gas
  • An english-spanish flashcard
  • The current time

What do all these have in common? It's simple: they are contain information. Data is just another word for information. By itself, data doesn't do anything. Data is simply information that you can use in your code.

On the flip side, code is commands we give to the computer. For example, take the following code:

var name = 'Bob';

AND and OR

You can evaluate multiple conditions at once using the AND and OR operators. AND is written as &&. OR is written as ||:

1
2
3
4
5
6
7
8
 
// JavaScript
var number = 8;
if (number >= 1 && number <= 100) {
  alert('The number is within 1 and 100');
}
if (number < 1 || number > 100) {
  alert('The number is NOT 1-100!');
}

When using &&, both conditions must be met, while only one must be met when using ||.

Sameness in JavaScript

In JavaScript, there is a difference between == and ===.

'17' == 17, but '17' !== 17.

Double equals will compare strings and integers, triple equals will recognize that these are not the same. For practical purposes, you should generally use triple equals, as its effects will be more predictable.

Truthy && Falsey Values

JavaScript has what is called "truthy" and "falsey" values. Some values evaluate to 'true', such as 2 === 2. and some values evaluate to false, such as 2 === 3.

JavaScript also has what is called truthy && falsey values. These are values that aren't strictly speaking 'true' or 'false', but instead evaluate to true or false in certain situations. 1 isn't true in and of itself, but it is 'truthy', that is to say, it evaluates to true in certain situations.

Put the following in your console:

1
2
3
4
5
6
 
if (1) {
  console.log("Truthy")
}
else {
console.log("Falsey")
}

The code prints out "truthy" because 1 is a "truthy" value. If you tried the same thing with 0, it would print out 'falsey', because 0 is a falsey value.
