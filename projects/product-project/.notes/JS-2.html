String Properties

Take the following code:

1
2
3
4
 
var house = {
  price: 90000
};
house['price'] //=> 90000

We see that we are creating an object with one property. Key point: The property is created as price:, but it is accessed via 'price'. Property names are (almost) always created as strings.

In other words, this is shorthand for:

1
2
3
4
 
var house = {
  "price": 90000
};
house['price'] //=> 90000

In fact, for JSON, you are required to use quotes for property names. However, when writing normal JavaScript code, you can just use the shorthand version.

JavaScript Arrays

JavaScript arrays are just special-treated objects. They have only a few special attributes that allow them to be used like arrays in other languages.

Iterating Through Arrays

JavaScript takes a very traditional approach to looping through arrays:

1
2
3
4
 
var names = ['Alice', 'Bob'];
for (var i=0; i < names.length; i += 1) {
  alert('Hello ' + names[i]);
}

In a for loop, there are three statements, separated by semicolons:

  1. Variable Initialization - This is where you put the variable you will use to keep track of how many times you have executed the loop.
  2. Loop Conditional - As long as this conditional is true, the loop will continue looping.
  3. Increment Variable - This statement happens right after a single loop executes. Usually it just increments the variable you declared in statement 1.

If you don't need to modify how you are incrementing through the array, you can also use the JavaScript forEach method.

1
2
3
4
 
var names = ['Alice', 'Bob'];
names.forEach(function(v,i,o) {
  alert('Hello' + v)
})

v represents the element of the array, i represents the current index of the array, and o represents the original array.

JavaScript Function Basics

Let's look at how you declare a function and then call it:

1
2
3
4
 
var sayHello = function (name) {
  alert("Hello, " + name);
};
sayHello('Mr. President');

To return a value, you must explicitly use the return keyword:

1
2
3
4
 
var square = function (number) {
  return number * number;
};
alert('12 squared is: ' + square(12));

In Javascript, you always need parentheses when calling a function, even when there are no arguments:

1
2
3
4
 
var theAnswer = function () {
  return 42;
};
theAnswer();

Notice how declaring a function looks very similar to declaring a variable. Keep that in mind for future lessons ;)

Parameters and Arguments

Did you see number in the function (number) example above? This is called a parameter. Parameters are basically placeholders for what a function wants.

For example, the following function multiply has two parameters:

1
2
3
4
5
 
var multiply = function (x, y) {
  return x * y;
};

multiply(3, 4);

In the above example, x and y are parameters. You can name parameters anything you want. It doesn't matter what you call them; you could rename x to potato if you really wanted to.

There's another side to this coin. In line 5, 3 and 4 are called arguments. Arguments are the things that you feed a function. In programmer speak, we are passing 3 and 4 as arguments to the multiply function call.

Explicit Returns

In ruby, methods return the value of the last line of code by default. In JavaScript, functions return undefined by default. If you ever want to send data back, you must use the return statements.

Here is the wrong way:

1
2
3
4
5
6
 
var mult = function(x, y) {
  x * y;
}

var res = mult(5, 10);
console.log(res); // This will result in undefined being printed

Here is the right way:

1
2
3
4
5
6
 
var mult = function(x, y) {
  return x * y;
}

var res = mult(5, 10);
console.log(res); // This will result in 50 being printed
