Product Project : Directions

Overview

You've been given some product data in the file located at data/product.json, and from this dataset, you're going to create a page of searchable and filterable products! You can style the project and layout the products as you see fit.

Have a look at the data available data/product.json - this will give you an idea of the ways in which a user can search or filter on products by type, search for products by description keyword queries, order by cost, and maybe build in other features, like if there's less than 10 products, show "Hurry, only x left in stock!". The page should feel like an Amazon product search experience.

You'll want to make use of forms and layouts, and of course, your functional programming chops! Below we've listed a set of review-resources to help you in your work.

Load the Product Data

  1. We need to first load our data before we can display it in any way: our product data is located in a file at data/product.json. We can use the $.getJSON() method to load our data. Google it, or find examples from our previous projects.

Display the List of Product

  1. Now that we have our data loaded, we need to display it in some fancy way to our user. Make us of jQuery to build a list of product! First thing to do would be to get the data on the page, next, make it pretty! Review all the material below for ideas!
  2. Note that there are also product thumbnails and full size images within the img/product directory! The file names of the images are referenced in the product.json file as, image, so you should use them in your product profiles.
  3. Finally, because you have both a thumbnail and full size image for the products, both named the same with respective thumbnails in the directory img/product/thumbs, you'll receive big bonus points if you design you product project to show product details when a user clicks on a product within the list.

Search & Filter Features

Just like search for crap on Amazon, you want to allow your user to execute keyword searches or fitler by product type, or lowest to highest cost. Go nuts here, you have lodash installed in your client project, so you can make use of all its functional goodies as you would with underscore.js.
